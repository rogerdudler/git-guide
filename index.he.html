יצירת מאגר חדש

ליצור תיקייה חדשה, להיכנס אליה ולבצע את הפקודה
git init
כדי ליצור מאגר git חדש.

יצירת עותק של מאגר

כדי ליצור עותק עבודה של מאגר מקומי יש להריץ את הפקודה
git clone /path/to/repository
כאשר משתמשים בשרת מרוחק, הפקודה תהה
git clone username@host:/path/to/repository

זרימת עבודה

המאגר המקומי שלך מורכב משלושה "עצים" המתוחזקים על־ידי git. הראשון הוא Working Directory המחזיק את הקבצים עצמם. השני הוא ה־Index שמתפקד בתור אזור ההערכות. השלישי והאחרון הוא HEAD המצביע על השמירה האחרונה שבוצעה.

הוספה ושמירה

ניתן להציע שינויים (ולהוסיף אותם אל ה־Index) באמצעות
git add <filename>
git add *
זהו הצעד הראשון בזרימת העבודה הפשוטה של git. כדי לשמור את השינויים שביצעת יש להשתמש בפקודה
git commit -m "Commit message"
כעת הקובץ נשמר ב־HEAD, אבל עדיין לא במאגר המרוחק.

דחיפת שינויים

השינויים שביצעת נמצאים כעת ב־HEAD של תיקיית העבודה המקומית שלך. כדי לשלוח אותם אל המאגר המרוחק עליך לבצע את הפקודה
git push origin master
יש להחליף את master בענף אליו נרצה לשלוח את השינויים שביצענו.

אם לא יצרת עותק של של מאגר קיים וברצונך להתחבר אל מאגר בשרת מרוחק, עליך להוסיף אותו עם הפקודה
git remote add origin <server>
כעת נוכל לדחוף את השינויים לשרת המרוחק הנבחר.

יצירת ענפים

ענפים משמשים כדי לפתח תכונות באופן נפרד מהאחרות. הענף master הוא ענף "בררת המחדל" כאשר יוצרים מאגר. נהוג להשתמש בענפים אחרים לפיתוח ולמזג אותם חזרה לענף הראשי בהשלמת העבודה.

כדי ליצור ענף חדש בשם "feature_x" ולעבור אליו יש לבצע
git checkout -b feature_x
כדי לעבור חזרה לענף master
git checkout master
וכדי למחוק את הענף החדש יש לבצע
git branch -d feature_x
הענף אינו זמין לאחרים כל עוד לא דחפנו אותו למאגר המרוחק.
git push origin <branch>

עדכון ומיזוג

כדי לעדכן את המאגר המקומי עם השמירות האחרונות, יש לבצע
git pull
בתיקיית העבודה כדי למשוך ולמזג remote את השינויים.
כדי למזג ענף אחר לענף הפעיל (למשל master) יש לבצע
git merge <branch>
בשני המקרים git ינסה לבצע מיזוג אוטומטי לשינויים. לצערנו, לא תמיד הדבר מתאפשר ודבר זה גורם לעתים לקונפליקטים. האחריות למיזוג ידני של קונפליקטים אלו נופלת עליך באמצעות עריכת הקבצים המוצגים על־ידי git. לאחר השינויים עליך לסמן אותם כממוזגים באמצעות
git add <filename>
לפני מיזוג שינויים, ניתן לקבל תצוגה מקדימה שלהם באמצעות
git diff <source_branch> <target_branch>

תיוג

מומלץ ליצור תגיות להוצאת גרסאות תוכנה. מדובר ברעיון מוכר, שנמצא בשימוש גם ב־SVN. ניתן ליצור תגית חדשה בשם 1.0.0 באמצעות הפקודה
git tag 1.0.0 1b2e1d63ff
הקוד 1b2e1d63ff מתייחס לעשרת התווים הראשונים במזהה השמירה שרוצים להתייחס אליו עם התגית. ניתן לקבל את מזהה השמירה עם הפקודה
git log
ניתן גם להשתמש בפחות תווים ממזהה השמירה, אבל עליו להיות ייחודי.

החלפת שינויים מקומיים

במקרה שעשית משהו שלא כשורה (דבר שכמובן לעולם אינו קורה ;) ניתן להחליף את השינויים המקומיים עם הפקודה
git checkout -- <filename>
פקודה זו מחליפה את השינויים בעץ העבודה שלך עם התוכן האחרון מ־HEAD. לא ישתנו שינויים שכבר התווספו אל ה־index, כמו גם קבצים אחרים.

כדי למחוק את כל השינויים המקומיים והשמירות, ולמשוך את ההיסטוריה העדכנית מהשרת ולכוון לשם את ענף master המקומי, יש לבצע זאת כך:
git fetch origin
git reset --hard origin/master

טיפים שימושיים

הממשק הגרפי המובנה של git
gitk
שימוש בפלט צבעוני
git config color.ui true
הצגת הרישום עם שורה בודדת עבור כל שמירה
git config format.pretty oneline
הוספת קבצים אינטראקטיבית
git add -i
