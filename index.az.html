Qurmaq

Windows üçün git'i Yüklə

Linux üçün git'i Yüklə

yeni bir depo yaratmaq

yeni bir depo yaratmaq üçün, bir qovluq yaradıb içərisində
git init
funksiyanı çalıştırın.

bir deponu kopyalamaq

lokal deponuzun işləyən bir surəti yaratmaq üçün
git clone /yol/depo
uzaq server istifadə etdiyiniz vəziyyətdə
git clone istifadəçiadi@server:/yol/depo

iş axışı

lokal deponuzun git tərəfindən idarə olunan üç "ağaçdan" ibarət olur. birincisi real faylları tutan əməliyyat kataloğu. ikinci etap Stage və nəticəsini etdiyiniz ən son commit'i göstərən HEAD.

əlavə & təslim

Dəyişiklikləri qeyd etmək (Index'e əlavə etmək) üçün
git add <fayladı>
git add *
Əsas git iş axışında bu ilk adımdır. Dəyişiklikləri depoya əlavə etmək üçün
git commit -m "Təslim mesajı"
artıq fayllar HEAD'e əlavə edildi, lakin hələ uzaq deponuza deyil.

dəyişiklikləri göndərmək

Artıq dəyişiklikləriniz lokal deponuzun HEAD'i içərisində.
Bu dəyişiklikləri uzaq deponuza göndərmək üçün,
git push origin master
dəyişiklikləri uzaq deponuzdakı göndərmək istədiğiniz filial'ı master ilə dəğiştirin.

Hələ uzaq bir deponuz yoxsa və uzaq depo əlavə etmək istəyirsinizsə,
git remote add origin <server>
Artıq dəyişikliklərinizi uzaq serverə göndərəbilərsiniz

filiallar ilə işləmək

Filiallar fərqli xüsusiyyətləri ayrı-ayrı inkişaf etdirmək üçün istifadə edilər. Yeni bir depo yaratdığınızda master "standart" filialdır. Digər filiallar inkişaf etdirdiktən sonra master'ə birləşdirilir.

"feature_x" adıyla yeni bir filial meydana gətirib o filiala keçmək üçün
git checkout -b feature_x
master'i bərpa etmek üçün
git checkout master
və yaratdığımız filialı silmək üçün
git branch -d feature_x
bir filialı uzaq deponuza göndərmədikçe
başqaları tərəfindən istifadə edilə bilinməz
git push origin <filial>

yenilemə & birləşdirmə

ən son dəyişiklikləri (commit) lokal deponuza əlavə etmək üçün
git pull
funksiyanı çalıştırın. Bu dəyişiklikləri al (fetch) ve birləştir (merge) edəcəktir. Aktiv filiala (məsələn master) başqa bir filialı birləştirmək üçün
git merge <filial>
hər iki vəziyyetdə də git dəyişiklikləri avtomatik birləştirməyi (auto-merge) yoxlar. Təəssüf ki, bu hər zaman mümkün olmaya bilər və toqquşmalarla (conflict) nəticələnər. Git tərəfindan göstərilən fayllari əl ilə düzəldərək bu toqquşmaları birləştirmək sizə düşər. Dəyişikliklərdən sonra, faylları əlavə etmək üçün
git add <fayladı>
dəyişiklikləri birləştirmədən öncə, önizləmə yapmaq üçün
git diff <mənbə_filial> <hədəf_filial>

Versiyalaştırmaq

proqram təminatı versiyalarınız üçün versiya adı (tag) yaratmağınız tövsiyə edilir. Bu SVN'də də mövcut olan bilindik bir anlayışdır. 1.0.0 adıyla bir versiya nömrəsini (tag) yaratmaq üçün
git tag 1.0.0 1b2e1d63ff
buradakı 1b2e1d63ff nümayiş olunacaq proqram versiyanızın əməliyyat nömrəsinin ilk 10 obrazıdır. Əməliyyat identifikasiya nömrələrini görmək üçün
git log
tək olduğu müddətcə daha az əməliyyat nömrəsi də istifadə edə bilərsiniz.

lokal dəyişiklikləri bərpa etmək

Səhv bir şey etdiyiniz vəziyyətində (təbii ki belə şeylər heç olmaz;)) lokal dəyişikliklərin geri almaq üçün
git checkout -- <fayladı>
bu dəyişikliklərinizi HEAD içərisindəki son məzmun ilə dəyişdirər. Index'ə əvvəlcədən əlavə olunmuş dəyişikliklər və yeni fayllar qorunacaq.

Əgər bütün lokal dəyişiklik və teslimlerinizi ləğv etmək istəyirsinizsə, serverdən ən son qeydləri gətirin və lokal master dalınıza göstərin
git fetch origin
git reset --hard origin/master

faydalı ipuçları

daxili git GUI (Qrafik İstifadəçi İnterfeysi)
gitk
rəngli git çıxışı istifadə edin
git config color.ui true

git config format.pretty oneline
interaktiv olmuş istifadə
git add -i
