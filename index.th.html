สร้าง repository ใหม่

สร้าง directory ใหม่ เปิดเข้าไปในนั้น แล้วใช้คำสั่ง
git init
เพื่อสร้าง git repository ใหม่

ทำสำเนาจาก repository อื่น

ทำสำเนาจาก repository ภายในเครื่องเดียวกัน ด้วยคำสั่ง
git clone /path/to/repository
ถ้าจะทำสำเนาจากเครื่องอื่นในเน็ทเวอร์ค ให้ใช้คำสั่ง
git clone username@host:/path/to/repository

ขั้นตอนการทำงานพื้นฐาน

สำเนาของ repository ที่อยู่ในเครื่องของคุณ ซึ่ง git ดูแลนั้นประกอบด้วย
"ต้นไม้ข้อมูล" 3 ต้น
ต้นแรกคือ Working Directory ซึ่งเป็นที่อยู่ของไฟล์ต่างๆ
ต้นที่สองคือ Index ซึ่งทำหน้าที่เป็น พื้นที่เตรียมงาน (staging area) และ
สุดท้ายคือ HEAD ซึ่งเป็นตัวชี้ตำหน่ง commit ล่าสุดของคุณ

เพิ่ม & บันทึก

คุณสามารถนำการเปลี่ยนแปลงต่างๆเพิ่มเข้าไปใน Index ได้ด้วยสำสั่ง
git add <filename>
git add *
นี่คือขั้นตอนพื้นฐานของการทำงานกับ git และเมื่อต้องการจะทำการบันทึกความเปลี่ยนแปลงเหล่านั้น ให้ใช้คำสั่ง
git commit -m "Commit message"
ตอนนี้ไฟล์ก็ได้รับการบันทึก และถูกชี้โดย HEAD ในเครื่องของคุณ โดยที่ยังไม่ส่งผลต่อ repository ที่ตำแหน่งอื่นแต่อย่างใด

ส่งต่อการเปลี่ยนแปลง

ตอนนี้การเปลี่ยนแปลงต่างๆยังอยู่ที่ HEAD ในเครื่องของคุณเท่านั้น เพื่อที่จะส่งต่อการเปลี่ยนแปลงเหล่านั้นไปที่ repository ที่ตำแหน่งอื่น ให้ใช้คำสั่ง
git push origin master
โดยแทนที่ master ด้วยชื่อของ branch ที่คุณต้องการที่จะส่งต่อการเปลี่ยนแปลงไปบันทึก

ถ้าคุณไม่ได้เร่ิมจากการทำสำเนาจาก repository อื่นที่มีอยู่แล้ว และต้องการที่จะเชื่อมต่อ repository ของคุณเข้ากับเครื่องอื่นในเน็ทเวอร์ค คุณต้องเพิ่มตำแหน่งของเซอร์เวอร์เข้าไปก่อน ด้วยคำสั่ง
git remote add origin <server>
ตอนนี้คุณก็สามารถส่งต่อการเปลี่ยนแปลงของคุณไปยังเซอร์เวอร์ตามที่กำหนดไว้ได้

แตกสาขา

เราใช้การแตกสาขาหรือ branch เพื่อให้สามารถแยกพัฒนาฟีเจอร์ต่างๆได้โดยอิสระจากกันได้ โดยมี master เป็น branch ที่ถูกกำหนดขึ้นโดยปริยายเมื่อคุณสร้าง repository ขึ้นมา
คุณควรจะสร้าง branch อื่นขึ้นมาเพื่อใช้ในการพัฒนา แล้วจึง merge พวกมันเข้ากับ master เมื่อพัฒนาฟีเจอร์นั้นๆเสร็จแล้ว

สร้าง branch ใหม่ในชื่อ "feature_x" และเปลี่ยนไปใช้ branch นั้นด้วยคำสั่ง
git checkout -b feature_x
เปลี่ยนกลับไปใช้ master
git checkout master
และลบ branch นั้นทิ้งไป
git branch -d feature_x
branch ที่คุณสร้างขึ้นนั้น จะยังคง ไม่ปรากฏต่อผู้อื่น จนกระทั่งคุณได้ทำการส่งต่อมันไปที่เซอร์เวอร์ ด้วยคำสั่ง
git push origin <branch>

ปรับปรุง & ควบรวมสาขา

ปรับปรุง repository ที่เครื่องของคุณให้ทันสมัย รวมถึงความเปลี่ยนแปลงต่างๆจนถึง commit ล่าสุด ด้วยการใช้คำสั่ง
git pull
ใน directory ที่คุณใช้งานเพื่อ ดึง และ ควบรวม ความเปลี่ยนแปลงล่าสุดจากเครื่องเซอร์เวอร์
เมื่อต้องการควบรวม branch อื่นเข้ากับ branch ปัจจุบัน (เช่น master) ให้ใช้คำสั่ง
git merge <branch>
git จะพยายามประสานความเปลี่ยนแปลงต่างๆให้โดยอัตโนมัติในทั้งสองกรณี แต่ถ้าหากว่ามี conflicts เกิดขึ้น คุณจะต้องเข้าไปจัดการความขัดแย้งเหล่านั้นด้วยการ แก้ไขเนื้อหาในไฟล์ต่างๆที่ git แจ้งว่ามีปัญหาด้วยตนเอง และเมื่อทำการแก้ไขแล้ว ให้ใช้คำสั่ง
git add <filename>
เพื่อแจ้งต่อ git ว่าคุณได้จัดการกับความขัดแย้งเหล่านั้นเป็นที่เรียบรอยแล้ว
ก่อนที่จะทำการรวม branch คุณยังสามารถตรวจดูความแตกต่างระหว่าง branch ต่างๆก่อนได้ ด้วยคำสั่ง
git diff <source_branch> <target_branch>

แปะป้าย

เราแนะนำให้แปะป้ายด้วยการสร้าง tag สำหรับทุกๆครั้งที่มีการส่งมอบซอฟแวร์เกิดขึ้น มันเป็นคอนเซปที่ยอมรับกันโดยทั่วไปและมีปรากฏอยู่ใน SVN ด้วยเข่นกัน คุณสามารถสร้างและตั้งชื่อป้ายว่า 1.0.0 ได้ด้วยคำสั่ง
git tag 1.0.0 1b2e1d63ff
โดย 1b2e1d63ff คือตัวอักษร 10 ตัวแรกของ commit id ที่คุณต้องการที่จะอ้างถึงด้วยป้ายของคุณ คุณสามารถหา commit id ด้วยการดูจาก...

ประวัติความเปลี่ยนแปลง

คุณสามารถศึกษาประวัติความเปลี่ยนแปลงของ repository ด้วยวิธีการที่ง่ายที่สุดคือ... git log
คุณอาจเพิ่ม parameter ต่างๆต่อท้ายเพื่อให้ได้รูปแบของประวัติความเปลี่ยนแปลงตามที่คุณต้องการ หากต้องการเห็นเพียง commit เฉพาะจากผู้ใช้รายใดรายหนึ่ง:
git log --author=bob
หากต้องการเห็นประวัติความเปลี่ยนแปลงในรูปแบบที่กระชับ เพียงหนึ่งบรรทัดต่อ commit:
git log --pretty=oneline
หรือคุณอาจต้องการเห็นต้นไม้ข้อมูลที่แสดง branch ทั้งหมด ตกแต่งด้วยชื่อของ tag และชื่อของ branch ต่างๆ ในรูปแบบของ ASCII อาร์ท:
git log --graph --oneline --decorate --all
ต้องการดูเฉพาะไฟล์ที่มีการเปลี่ยนแปลง:
git log --name-status
ทั้งหมดนี้เป็นเพียงตัวอย่างเล็กน้อยของ parameter ที่คุณสามารถเลืิอกใช้ได้ ซึ่งคุณสามารถศึกษาเพิ่มเติมได้ที่ git log --help

ยกเลิกการเปลี่ยนแปลงที่เครื่องของคุณ

ในกรณีที่คุณทำผิดพลาดบางอย่าง ซึ่งก็ไม่เคยเกิดขึ้นอยู่แล้ว ;) คุณสามารถ แทนที่ส่ิงที่คุณได้แก้ไขไปแล้ว ด้วยคำสั่ง
git checkout -- <filename>
ซึ่งจะแทนที่ความเปลี่ยนแปลงที่ของคุณใน Working Directory ด้วยเนื้อหาล่าสุดใน HEAD โดยความเปลี่ยนแปลงต่างๆที่ถูกเพิ่มเข้าไปแล้วใน Index รวมถึงไฟล์ใหม่จะยังคงอยู่ ไม่หายไปไหน

แต่หากว่าคุณต้องการที่จะทิ้งความเปลี่ยนแปลงและ commit ทั้งหมดที่เกิดขึ้นที่เครื่องของคุณซะ แล้วดึงเอาเนื้อหาล่าสุดจากเซอร์เวอร์ และกำหนดให้ master ที่เครื่องของคุณให้ชี้ไปด้วย ก็ให้ทำแบบนี้
git fetch origin
git reset --hard origin/master

ข้อแนะนำที่มีประโยชน์

ส่วนติดต่อกับผู้ใช้แบบกราฟฟิกส์ที่ติดตั้งมาด้วย
gitk
กำหนดให้ใช้สีในการแสดงผล
git config color.ui true
แสดง log ของแต่ละ commit ในบรรทัดเดียว
git config format.pretty oneline
ใช้วิธีเพิ่มไฟล์แบบมีส่วนร่วมทีละขั้นตอน
git add -i
